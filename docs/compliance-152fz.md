# Обработка контактов и открытые источники

> Это операционная памятка для менеджеров и разработчиков, **не юридическая консультация**. Перед продакшеном согласуйте текст с юристом.

## Принципы

1. **Только открытые источники** — публичные сайты, материалы СМИ, рейтинги, страницы мероприятий. Не автоматизируем обход закрытых соцсетей и личных кабинетов.
2. **Цель** — B2B-продажи FeedBackTalk (CX / HR / исследования), минимизация данных: только поля, нужные для контакта.
3. **Прозрачность** — в карточке контакта ведётся журнал происхождения (`data_provenance_log`): откуда взяты e-mail/телефон.
4. **Верификация менеджером** — перед коммерческим письмом нажмите «Канал проверен» (`channel_verified_at`).
5. **Срок хранения** — рекомендуем пересмотр/удаление неактуальных карточек через **24 месяца** (настраивается политикой компании).

## Капча и блокировки

Если поисковик отдаёт капчу, задача переходит в статус `needs_captcha`. Менеджер открывает ссылку в браузере, проходит проверку и загружает HTML или cookies — исследование продолжается.

Опционально: `CAPTCHA_SERVICE_URL` + `CAPTCHA_API_KEY` в `.env` (интеграция не привязана к конкретному вендору).

## Запрещённые домены (авто)

По умолчанию не сохраняем происхождение с facebook.com, instagram.com, tiktok.com.
